# 插件 platform.yaml 参考

## 文件用途

`platform.yaml` 字段 **与 QQ 开放平台表单一一对应**，开发者打开官网后直接复制填写。

官网路径：`q.qq.com` → 机器人 → **功能配置**

## 服务配置表单字段

| 官网字段 | platform.yaml 键 | 说明 |
|----------|------------------|------|
| ID | `ID` | 自动或手动序号 |
| 名称 | `名称` | ≤5 字，中英数字 |
| 介绍 | `介绍` | ≤15 字，中英数字 |
| appID | `appID` | 小程序 ID（非机器人 AppID） |
| path | `path` | 小程序页面路径 |
| extendData | `extendData` | 群聊留空；频道用 `$OPENGUILDID$` |
| 权限菜单 | `权限菜单` | `所有人` 或 `频道主和管理员` |
| 使用场景 | `使用场景` | 列表：`QQ频道` `频道私信` `QQ群` `消息列表` |

纯文字插件写：`服务配置: 无需配置`

## 指令配置表单字段

| 官网字段 | platform.yaml 键 | 说明 |
|----------|------------------|------|
| ID | `ID` | 序号 |
| 指令名 | `指令名` | **不带 `/`**，官网自动显示为 `/xxx` |
| 指令介绍 | `指令介绍` | ≤15 字 |
| 权限菜单 | `权限菜单` | `所有人` |
| 使用场景 | `使用场景` | 同服务配置 |
| 是否传参 | `是否传参` | `否` 或 `是` |

## 完整模板

见 `plugins/_template/platform.yaml`

## 汇总清单

所有插件合并后的填写表见 `docs/官网配置清单.md`

## 交付用户格式

新增插件后，必须：

1. 更新 `plugins/{name}/platform.yaml`
2. 在 `docs/官网配置清单.md` 追加对应表格行
3. 告知用户：服务配置是否需填、指令配置逐条复制
